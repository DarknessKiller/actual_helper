# Multi-Provider Cleanup Design

## Goal

Make the codebase ready to add a second bank/fintech provider without modifying shared infrastructure code. This is purely a refactoring and hygiene pass — no new features.

## Changes

### 1. Bootstrap: Accept Provider Factories

**File:** `internal/bootstrap/bootstrap.go`

Current `Init()` hardcodes `tngprov.New(...)`. Change to accept a map of provider factories:

```go
type ProviderFactory func(excludeKeywords, includeKeywords []string, categories []models.CategoryRule) providers.Provider

func Init(factories map[string]ProviderFactory) (*providers.Registry, *config.Loader)
```

Bootstrap iterates the map, calls `loader.ProviderConfig(name)` for each, passes the merged rules to the factory, and registers the provider. No config path → log warning, still iterate factories with empty rules.

`main.go` wires providers:
```go
registry, loader := bootstrap.Init(map[string]bootstrap.ProviderFactory{
    "tng": tngprov.New,
})
```

Adding a new provider in the future requires one line in `main.go` + one new package — no changes to bootstrap.

### 2. Extract `internal/rule/` Package

**New files:**
- `internal/rule/engine.go`
- `internal/rule/engine_test.go`

Keyword filtering (`shouldSkip`) and category matching (`matchCategory`) are completely generic patterns that every provider with config rules will need. Extract to a shared `rule.Engine`:

```go
type Engine struct {
    excludeKeywords []string
    includeKeywords []string
    categories      []models.CategoryRule
    mu              sync.RWMutex
}

func NewEngine(exclude, include []string, cats []models.CategoryRule) *Engine
func (e *Engine) Reload(exclude, include []string, cats []models.CategoryRule)
func (e *Engine) ShouldSkip(description string) bool
func (e *Engine) MatchCategory(description string) (group, category string)
```

Methods own the mutex — safe for concurrent use. TNGProvider replaces its own keyword fields + mutex with an embedded `*rule.Engine`.

`TNGProvider.New()` still takes the same parameters — it passes them to `rule.NewEngine()`. Constructor signature stays compatible with `ProviderFactory`.

Test the engine directly (table-driven, all cases + concurrent safety).

### 5. Handler Test: Use Mock

**File:** `internal/handlers/convert_test.go`

Replace `tngprov.New(nil, nil, nil)` with a registered mock provider. This decouples the handler test from any specific provider implementation.

Create an inline mock that returns an empty report list for any CSV/PDF input. The mock doesn't need a new file — it lives in the test file.

### 6. TNG Provider: Internal Simplification

**File:** `internal/providers/tng/service.go`

- Remove `excludeKeywords`, `includeKeywords`, `categories` fields and their `mu`
- Replace with embedded `*rule.Engine`
- `shouldSkip` and `matchCategory` delegate to `e.ShouldSkip()` / `e.MatchCategory()`
- `Reload` delegates to `e.Reload()`
- Constructor copies to engine as before

The public API (`New`, `Reload`, `Name`, `ParseCSV`, `ParsePDFText`) stays identical. `ProviderFactory` signature works unchanged.
